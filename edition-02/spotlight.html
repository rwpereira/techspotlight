🔐 feature #11765 ext-integration.ts
Portal de API externa com documentação rica — Aprosoja
❌ problema
Parceiros externos (ex: Aprosoja) precisavam cadastrar produtores e propriedades no Agrotrace via API, mas não havia um canal seguro, documentado e autossuficiente para isso.
✓ solução
Portal público com tela de login por token JWT, seguido de documentação completa da API — mais rica que Swagger — com exemplos de payload, lógica de upsert e detalhes de autenticação.
CMS gera token
→
Parceiro acessa portal
→
Cola token JWT
→
Lê documentação
→
Integra via API

📡 feature #11825 + #11910 health-dashboard.ts
Painel de Monitoramento do sistema — CMS
❌ problema
Não havia visibilidade de erros de integração em tempo real. Debugging era feito no escuro — sem histórico, sem agrupamento por hora, sem contexto da requisição que falhou.
✓ o que foi construído
→ Gráfico de erros em linha do tempo contínua de 24h, ancorada no horário atual
→ Modal com body JSON completo + query params de cada requisição
→ Dashboard com tabs por categoria: logs de sucesso, logs de erro, insights
→ Scrollbar customizada no visualizador JSON
📈 resultado
Visibilidade imediata da saúde da API sem precisar de ferramentas externas — erros agrupados, contexto completo e histórico de 24h a um clique.

// lightning — Bruno — 5 min
⚡ ferramenta interna open-source pr-tools.sh
PR Tools — Descrições de PR e Test Cases com IA
❌ antes
Escrever descrição de PR manualmente: copiar card do Azure DevOps, descrever mudanças, montar checklist de testes. Para Test Cases, criar o card no board, preencher cenários e passos. ~15 min por PR + ~10 min por Test Case.
✓ agora — dois comandos
create-pr-description
Coleta git diff + git log, detecta sprint vigente e work item da branch, envia para LLM e gera título + descrição formatada. Cria o PR no Azure DevOps automaticamente com reviewers e work items vinculados.
create-test-card
Detecta PR e work item da branch atual, lê as mudanças do PR e gera Test Case completo (Objetivo, Cenário Base, Checklist, Resultado Esperado). Cria automaticamente o card no Azure DevOps como filho do work item.
~25
min antes (PR + TC)
→
~2
min agora
$0
custo (modelos free)
Funcionalidades
✓ 3 providers LLM + fallback
✓ Detecta sprint vigente
✓ Cria PR via Azure API
✓ Cria Test Case via API
✓ Reviewers automáticos
✓ Work item da branch
✓ Clipboard automático
✓ Wizard de configuração
✓ Instala via curl one-liner
✓ macOS, Linux, WSL

📦 feature #11951 + #11950 rat-offline.html
Relatório de Atendimento 100% offline — index.html no ZIP
❌ problema
O ZIP do Relatório de Atendimento Técnico continha fotos e documentos soltos, sem contexto. O consultor em campo — muitas vezes sem internet — não tinha como visualizar o relatório de forma organizada.
✓ solução
Template HTML standalone incluído no ZIP via checkbox opt-in. O index.html funciona como um mini-site navegável com:
✓ Galeria de fotos com lightbox
✓ Lista de arquivos do relatório
✓ Toggle dark / light mode
✓ 100% offline — sem CDN
✓ Marcadores dinâmicos no template
✓ Responsivo (mobile/desktop)
📈 resultado
Consultor abre o ZIP, dá duplo-clique no index.html e navega tudo — fotos, documentos, dados do atendimento — sem internet, sem app, sem dependências.
