🤖 feature Novo Projeto agro-assistant.config.ts
Agro Assistant — IA para consultas agrícolas
❌ problema
Técnicos de campo precisam consultar informações agrícolas dispersas em documentos, manuais e protocolos. Não existe forma unificada de buscar esse conhecimento — consultas são lentas e dependem de quem conhece o assunto.
✓ solução
→ Monorepo com 3 apps: RAG (Python/FastAPI), API (NestJS/Drizzle), Web (React/Vite)
→ Serviço RAG com Docling + pgvector + sentence-transformers
→ Agente Mastra AI: classifica consulta → busca RAG → auto-reflexão → resposta
→ Módulo de fontes de conhecimento com upload, versionamento e retry
→ Docker Compose: PostgreSQL, Redis, MinIO

// spotlight 2 — Thielson — 5 min
⚡ feature #12029 respostas-automaticas.service.ts
Respostas Automáticas — de 100+ chamadas para 1
❌ problema
Ao editar uma pergunta com muitas respostas automáticas, o front fazia uma chamada ao backend para CADA item. Perguntas com 100+ respostas disparavam 100+ requests — tela travava, UX degradada, risco de timeout.
✓ solução
→ Refatorado para uma única chamada inicial que traz todos os itens
→ Chamadas individuais só ao adicionar ou editar um item específico
→ Adicionado contador visual para gestão
→ Filtro de perguntas inativas otimizado
📈 resultado
De 100+ chamadas HTTP para 1 no carregamento. Tela responsiva mesmo com centenas de respostas configuradas.

📊 feature #12062 carbon-dashboard.component.ts
Dashboard Carbono — KPIs e drill-down por tema
❌ problema
Programa Carbono sem visibilidade consolidada de KPIs. Dados dispersos em múltiplas tabelas, sem painel analítico para tomada de decisão. Consultas pesadas degradavam a performance.
✓ solução
→ Consulta SQL otimizada para extração performática dos dados
→ Dashboard com temas prioritários e KPIs visuais
→ Drill-down por tema para detalhamento granular
→ Gráficos interativos com filtros
📈 resultado
Painel consolidado com visão por tema, KPIs em tempo real e drill-down — substituindo consultas manuais por um dashboard self-service.

-- Query otimizada por tema
SELECT
  t.nome AS tema,
  COUNT(DISTINCT r.id) AS respostas,
  AVG(r.nota) AS media,
  SUM(CASE WHEN r.conforme
    THEN 1 ELSE 0 END) AS conformes
FROM tema t
JOIN resposta r ON r.tema_id = t.id
WHERE r.projeto_id = @projetoId
GROUP BY t.nome
ORDER BY media DESC
