🔐 feature #11765 ext-integration.ts
Integração externa via token JWT — Aprosoja
❌ problema
Parceiros externos (ex: Aprosoja) precisavam criar produtores e propriedades no Agrotrace, mas não existia nenhum endpoint seguro pra isso. Qualquer integração usava credenciais de usuário — inaceitável para uma API pública.
✓ solução
Módulo de integração dedicado com ExternalIntegrationGuard — JWT com prazo configurável e documentação integrada ao front.
CMS gera token
→
Guard valida JWT
→
Controller cria entidade
📈 resultado
módulo isolado JWT com validade doc embutida no front pronto para novos parceiros

@Injectable()
export class ExternalIntegrationGuard {
  async canActivate(ctx: ExecutionContext) {
    const token = extractBearer(ctx)
    const payload = await this.jwt.verify(token)
    if (payload.exp < Date.now())
      throw new UnauthorizedException()
    return true
  }
}

@UseGuards(ExternalIntegrationGuard)
@Post('pessoa-com-propriedade')
async criar(@Body() dto: CriarPessoaDto) {
  return this.svc.criar(dto)
}

📡 feature #11825 + #11910 health-dashboard.ts
Dashboard de saúde do sistema — CMS
❌ problema
Não havia visibilidade de erros de integração em tempo real. Debugging era feito no escuro — sem histórico, sem agrupamento por hora, sem contexto da requisição que falhou.
✓ o que foi construído
→ Gráfico de erros em linha do tempo contínua de 24h, ancorada no horário atual
→ Modal com body JSON completo + query params de cada requisição
→ Dashboard com tabs por categoria: logs de sucesso, logs de erro, insights
→ Scrollbar customizada no visualizador JSON
📈 resultado
Visibilidade imediata da saúde da API sem precisar de ferramentas externas — erros agrupados, contexto completo e histórico de 24h a um clique.
